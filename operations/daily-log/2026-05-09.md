## Strategic Decisions Made

### 1. Per-product admin separation (decision filed at `operations/decisions/2026-05-09-per-product-admin-architecture.md`)

Each product gets its own admin sub-tree at `/admin/<product-key>/*` with consistent sub-nav: Overview · Team · Users · Plans · Billing · CMS · Settings · Activity · Tutorials · Support. Brand color per product. URL prefix `pXX-<name>` makes ordering obvious.

URL restructure: `/admin/chatbase/*` → `/admin/p02-chatbase/*`; `/admin/taxpilot/*` → `/admin/p03-taxpilot/*` (git mv, history preserved).

### 2. Multi-app product split (decision filed at `operations/decisions/2026-05-09-multi-app-product-separation.md`)

Founder wants each product to be **fully extractable as an investor-sale asset.** Today's reality: all the code (P01/P02/P03/P04 admin + dashboard + APIs) lives in `products/01-claude-reseller/app/`. That's a soft separation, not a hard one.

Filed 6-phase migration plan:
- **Phase 0** (DONE today): npm workspaces config + `packages/` skeleton (9 placeholder shared packages) + per-product `app/README.md` placeholders.
- Phase 1 (next session): lift cross-product libs (rbac, audit, billing, cms, ai-support, tutorials, supabase) to `packages/`. Update P01 imports.
- Phase 2: extract P02 ChatBase to its own `products/02-whatsapp-ai-suite/app/` + own Vercel project.
- Phase 3-5: same for P03/P04, build P05/P06 fresh.
- Phase 6: per-product custom domains + per-product Clerk projects.

P01 deployed site is unchanged.

### 3. Tutorials with auto-translation
Founder's requirement: each product has video tutorials per feature, English + auto-translated audio + subtitles.

Pipeline: Groq `whisper-large-v3-turbo` for transcription → Claude Haiku 4.5 for VTT translation → multiple language tracks per video. Founder uploads English; system generates Hindi / Gujarati / Tamil / Telugu / Marathi / Bengali / Kannada / Malayalam / Punjabi.

Punted: TTS audio tracks (translated audio); chunked uploads >100MB.

### 4. Per-product AI support (separate from each other)
Each product has its own KB / conversations / messages tables (`<product>_support_*` × 6). Shared pure-functional engine in `lib/ai-support/` so when P02 is sold, the product's data extracts but the engine code travels too.

54 routes total: 12 public (rate-limited 30/hr per visitor+IP) + 42 admin (audit-logged).
