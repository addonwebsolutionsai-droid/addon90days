# P02 ChatBase — Required Environment Variables

## Existing (already on Vercel from P01 — no action needed)

| Variable | Used by |
|---|---|
| NEXT_PUBLIC_SUPABASE_URL | Supabase client |
| NEXT_PUBLIC_SUPABASE_ANON_KEY | Supabase public client |
| SUPABASE_SERVICE_ROLE_KEY | Supabase admin client (all P02 DB writes) |
| NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY | Clerk auth |
| CLERK_SECRET_KEY | Clerk server auth |
| GROQ_API_KEY | Intent classification + reply generation |

## New — must be added to Vercel before P02 routes work

| Variable | Required | Example | Notes |
|---|---|---|---|
| P02_ENCRYPTION_KEY | YES — all envs | `a3f2...` (64 hex chars) | AES-256-GCM key for storing WhatsApp access tokens. Generate: `openssl rand -hex 32` |
| META_WEBHOOK_VERIFY_TOKEN | Production only | `my-random-verify-string` | Match whatever you enter in the Meta App Dashboard > Webhooks > Verify Token |
| WHATSAPP_PHONE_NUMBER_ID | Production only | `123456789012345` | From Meta App Dashboard > WhatsApp > API Setup |
| WHATSAPP_ACCESS_TOKEN | Production only | `EAAx...` | System User token from Meta Business Manager |
| MOCK_MODE | Optional | `true` or `false` | Default is `true`. Set to `false` only when Meta verification is complete AND WHATSAPP_PHONE_NUMBER_ID is set |

## Steps to go from MOCK_MODE to production

1. Complete Meta Business Manager verification (founder action, 3-7 days).
2. Create a WhatsApp App at developers.facebook.com.
3. Note the Phone Number ID and generate a System User Access Token.
4. Add `WHATSAPP_PHONE_NUMBER_ID` and `WHATSAPP_ACCESS_TOKEN` to Vercel env.
5. Set `META_WEBHOOK_VERIFY_TOKEN` to a random string.
6. Configure the webhook URL in Meta App: `https://your-domain.com/api/webhook`
7. Set `MOCK_MODE=false` in Vercel (production target only).
8. In the Supabase dashboard: update `p02_workspaces.whatsapp_phone_number_id` for the workspace record.
9. Run the smoke test against the production URL with a real WhatsApp message.

## How to generate P02_ENCRYPTION_KEY

```bash
openssl rand -hex 32
```

Example output: `a3f2c1d4e5b6a7f8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a1b2`

This key must be the same across all Vercel environments (production + preview) or tokens encrypted in one env cannot be decrypted in another.

## Supabase note (IMPORTANT)

As of 2026-05-04, the Supabase project has **legacy API keys disabled**.
The `SUPABASE_SERVICE_ROLE_KEY` currently in `.env.local` is the old-format JWT key and
will return `{"message":"Legacy API keys are disabled"}`.

Founder action required:
1. Log in to Supabase Dashboard > Project Settings > API.
2. Either re-enable legacy keys OR copy the new-format `service_role` key from the new
   "JWT" section and update Vercel + `.env.local` with the new value.

The migration `010_p02_chatbase.sql` must be applied via the Supabase Dashboard SQL Editor
(Database > SQL Editor > New Query, paste the migration file contents, run).
