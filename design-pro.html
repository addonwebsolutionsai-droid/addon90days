Section 01 — Philosophy

The AddonWeb
Design Standard

Groundbreaking doesn't mean complex. It means the right information, at the right time, requiring the fewest possible decisions from the user. Every pixel earns its place.

⚡

Speed Over Beauty

A fast interaction users don't notice beats a beautiful one they admire but wait for. Render in <100ms or pre-load.

🎯

One Job Per Screen

Each view has exactly one primary action. Everything else is secondary. Users never wonder "what do I do here?"

🔁

Contextual Defaults

Smart defaults mean 80% of users never change a setting. Pre-fill, auto-detect, remember preferences.

🧠

Progressive Disclosure

Show only what the user needs right now. Advanced options live one click deeper — available but not in the way. 90% of power in 10% of screen.

Progressive Disclosure Example
Level 1: One-click "Create Invoice" button
Level 2: Invoice form (pre-filled from last order)
Level 3: Advanced → HSN override, multi-tax, custom terms
🌊

Flow State Design

Never interrupt a user mid-task with confirmation dialogs, upsells, or unrelated alerts. Protect their focus.

2025 Trends Applied — Godly.website Analysis

Trend 01 · Bento Grid

Asymmetric card grids replacing uniform layouts. Cards have different sizes (1×1, 2×1, 1×2) creating visual rhythm. Used on landing pages to showcase multiple features simultaneously without linear hierarchy.

Applied to: All 6 product landing pages
Trend 02 · Aurora Backgrounds

Animated gradient blobs behind hero sections. Blurred radial gradients with slow motion create depth. Colors are brand-specific, opacity kept at 10–20% so content remains readable.

Applied to: Hero sections + dashboard topbars
Trend 03 · Noise Texture

Subtle grain/noise overlay on dark backgrounds. Makes pure black feel organic and premium rather than flat. Applied via SVG feTurbulence filter at 3–5% opacity.

Applied to: All dark backgrounds globally
Trend 04 · Gradient Borders

Cards with gradient borders using CSS padding trick: background: linear-gradient → inner div covers it. More dynamic than solid borders. Used for highlighted/featured cards only — not everything.

Applied to: Hero stats, featured pricing tier
Trend 05 · Kinetic Typography

Animated gradient text on key headlines. CSS background-clip:text with gradient-shift animation. Used sparingly — hero H1 only. Rest of text is static for readability.

Applied to: Hero H1 of each product
Trend 06 · Glassmorphism

backdrop-filter:blur(20px) with rgba background on nav bars and floating elements. Creates depth hierarchy without heavy drop shadows. Requires non-flat background behind to show effect.

Applied to: Sticky navbars, command palette
Section 02 — Zero-Click UX

Work Done in
Fewest Clicks

Every AddonWeb product is designed around the principle that the fastest user is a happy user. Count the clicks before you ship the feature.

Click Budget — Max Allowed

Create + send an invoice ≤ 3 clicks
Add a WhatsApp auto-reply ≤ 2 clicks
See why a machine alert fired 0 clicks (visible on load)
Connect a new IoT device ≤ 4 clicks
Install a Claude Skill 1 click
File a GST return ≤ 5 clicks

7 Zero-Click Techniques

1. Command Palette (⌘K)
Jump anywhere in the app, trigger any action, search any record — without navigating. Required on every dashboard.
2. Inline Editing
Click any text field in a table or card to edit in place. No "edit mode" toggle. Auto-saves on blur.
3. Smart Pre-fill
Forms pre-fill from last session, GSTIN lookups, contact history. User confirms, not enters.
4. Floating Action Button (FAB)
One persistent button for the primary action on every page. Always reachable without scrolling.
5. Bulk Actions on Select
Selecting table rows reveals bulk action bar. Archive 50 invoices in one click, not fifty.
6. Keyboard Shortcuts
N = new, E = edit, D = delete, / = search, Esc = back. Power users work 3× faster.
7. Contextual Quick Actions
Hover a table row → action buttons appear inline. No separate "open record, scroll to action" flow.

Section 06 — Motion

Motion & Animation

Animation communicates state, not decoration. Every animation has a reason: guide attention, confirm action, show hierarchy, reduce perceived latency.

Animation Catalog

Fade-Up Entry duration: 400ms · ease-out
All cards/sections animate in from Y+20px, opacity 0→1. Triggered by IntersectionObserver at 20% visibility. Stagger: +50ms per child.
Gradient Shift duration: 4s · ease infinite
Hero H1 gradient text animates background-position 0%→100%→0% on 300% width gradient. Subtle, continuous, draws eye to headline.
Aurora Blob duration: 12–15s · ease infinite
Background gradient blobs translate + scale slowly. Creates depth without distraction. Opacity 10–15%, blur(80px).
Hover Lift duration: 150ms · ease
Cards translateY(-2px) + border-color brightens on hover. Tells user "this is interactive." Never skip on interactive cards.
Pulse Ping duration: 1s · ease infinite
Alert dots scale(1)→scale(2), opacity 1→0. Used ONLY for critical alerts, live status. Never decorative.
Shimmer Skeleton duration: 1.5s · ease infinite
Loading state: sweeping gradient left-to-right. Shows content shape before data arrives. Reduces perceived load time by 40%.
Count-Up Numbers JS · 1200ms · ease-out
KPI numbers animate from 0 to value when section enters viewport. Makes metrics feel live and impressive. JS requestAnimationFrame.

Section 07 — Anatomy

Landing Page Anatomy

10 sections. Fixed order. Each section has one job. Skipping or reordering drops conversion by 20–40%. Every AddonWeb product landing follows this exact sequence.

① Sticky Nav — Trust + Navigation
② Hero — Value prop above the fold
③ Logo Bar — Trust by association
④ Pain → Solution — Emotional hook
⑤ Bento Features — Rational proof
⑥ Testimonials — Social proof
⑦ Pricing — Commitment
⑧ FAQ — Handle objections
⑨ Final CTA — Last chance convert
⑩ Footer — Links + legal
① NAV — RULES
  • Logo left · 4–5 links center · 1 CTA button right. Always.
  • Sticky on scroll. Blur backdrop (backdrop-filter: blur(16px)). Background opacity 80% so content behind is visible.
  • CTA in nav = secondary. Hero CTA = primary. Same destination, different weight.
  • No dropdowns in nav unless absolutely required. Each dropdown = decision fatigue.
② HERO — THE MOST IMPORTANT SECTION
  • H1 ≤ 8 words. Active verb. Specific outcome. "Ship AI 10× faster" not "Empowering Businesses."
  • Product mockup or code snippet in hero — real UI, never abstract illustration
  • Primary CTA + ghost CTA. Primary = start trial. Ghost = see demo/docs.
  • 3 social proof stats below CTAs: user count, rating, revenue processed
  • Aurora gradient blob behind text for visual depth
④–⑤ PAIN/FEATURES — EMOTIONAL THEN RATIONAL
  • Pain card (red border) shows life WITHOUT product. Solution card (green border) shows WITH.
  • User thinks "that's exactly my problem" within 3 seconds
  • Features use bento grid, not bullet list. Cards are scannable, bullets are walls of text.
  • Hover on feature cards: brand accent border glow
⑦ PRICING — 3-TIER ALWAYS
  • 3 tiers: Starter · Growth (featured) · Enterprise. Middle always "Most Popular."
  • Annual toggle saves 20%. Show annual by default — higher ACV.
  • Enterprise = "Contact Sales." Never show enterprise price publicly.
  • Gradient border on featured tier. All other tiers: plain border.

Section 08 — Anatomy

Dashboard Anatomy

7 zones. Mofi-inspired density with AddonWeb dark aesthetic. KPIs always first. Charts always second. Table always third. Period toggles (W/M/Y) on every chart.

① Fixed Sidebar — Brand + grouped nav
② Sticky Topbar — Breadcrumb + actions + avatar
③ KPI Row — 3–4 metric cards with delta + mini sparkline
④ Chart Area — Line/bar main + donut secondary
⑤ Data Table — Sortable, filterable, bulk-selectable
⑥ Command Palette — ⌘K global, always
⑦ Notification Panel — Pinned alerts, activity feed
SIDEBAR RULES (from Mofi analysis)
  • Width: 200–220px fixed. Never collapse on desktop — it breaks muscle memory.
  • Grouped nav: Pinned favorites at top, main sections, settings at bottom
  • Badge counts (red pill) for unread alerts/notifications on nav items
  • Active item: brand-color bg at 12% + left 3px brand accent border
  • Logo click = go to dashboard. Never opens a menu.
KPI CARD RULES
  • Max 4 KPIs per row. Users can't scan more than 4 at a glance.
  • Each card: Label → Big Number → Delta (↑/↓ % vs period) → Mini sparkline or progress bar
  • Colored top border = category (green=revenue, red=alerts, blue=usage)
  • Period filter (W/M/Y) toggles apply to ALL cards simultaneously — one click changes context
TABLE RULES (Mofi-inspired)
  • Checkbox column for bulk select → bulk action bar slides up from bottom
  • Hover row → action icons appear inline right side (edit, delete, view)
  • Stacked avatars with +N overflow for team columns
  • Progress bar column for completion %, not just numbers
  • Type badge (pill) for categories. Never text-only for status.

RULE: STRICT BUTTON HIERARCHY

Gradient/Solid = primary (one per page). Ghost = secondary. Text = tertiary. Never two primary buttons at the same visual level. Never a red button for anything non-destructive. Disabled = not clickable + 40% opacity, always.
